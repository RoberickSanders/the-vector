# Example User — Scoring Rubric

You are scoring how strong a fit a job posting is for the candidate. Use this rubric and return a structured fit_score 1-10.

## Strong-fit signals (raise score)

- Roles matching the candidate's target list (edit this for your own targets):
  - GTM Engineer / Growth Engineer / RevOps Engineer / Applied AI Engineer / Forward Deployed Engineer
- Companies in the candidate's preferred verticals (AI tooling, dev infra, cybersecurity, etc.)
- Tech stack matches the candidate's resume (Python, SQL, Claude API, MCP, etc.)
- Pay band within target range (edit for your own band)
- Remote OR aligned to candidate's home metro

## Strong-fit candidate background (edit this section to describe yourself)

- ~N years revenue ops / GTM / engineering
- Specific role progression that demonstrates the trajectory
- Production systems shipped
- Education / certifications

## Hard NO signals (score ≤3)

- Hybrid / on-site outside candidate's home metro
- Pay below candidate's floor
- Commission-only or 1099-only roles
- Senior leadership roles requiring 15+ years experience

## Soft NO signals (score 4-6)

- Adjacent roles missing one or two key signals
- Director-level roles requiring direct-report management

## Fit score scale

- **9-10:** Posted JD with verified pay match, exact skill stack, location aligned. Apply this week.
- **7-8:** Strong fit, minor gap. Apply.
- **5-6:** Adjacent role with story-driven application potential. Stretch — apply only if low effort.
- **3-4:** Marginal fit. Skip unless something unusual makes it interesting.
- **1-2:** Wrong role, wrong location, or wrong pay band. Skip.
